# Life (Obsidian + Git)

Этот репозиторий — один Obsidian‑vault, который удобно:
- вести как базу заметок и знаний (Markdown + wiki‑ссылки `[[...]]`);
- хранить и ревьюить через Git (история, диффы, автоматизация);
- обслуживать любым AI‑агентом напрямую через файловую систему.

## Проблематика и цели
### Проблематика
- Входящие материалы (сообщения, голос, документы) копятся и теряются без единого процесса “захват → разбор → действие”.
- Задачи живут в разных местах и теряют контекст: сложно понять “почему это делаем” и “к какому проекту относится”.
- В больших списках задач растёт шум: сложно выделять главное и держать дедлайны под контролем.
- Без рутины обзоров система быстро “умирает”: планы не обновляются, связи рвутся, архив не ведётся.

### Цели
- Держать единый источник правды по работе/личному: **goal → project → tasks → notes** со связями `[[...]]`.
- Быстро превращать входящее (DOCX/PPTX/аудио) в заметки с `## Summary`, а затем — в задачи/проекты после уточнений.
- Иметь дашборды (Tasks/Dataview) и простую канбан‑доску для фокуса на ближайших сроках.
- Поддерживать аккуратный Git‑след: минимальный дифф, без удаления данных (архив вместо удаления), без секретов в репо.

### Агентный подход (роли, а не “один помощник”)
Этот vault рассчитан на работу с AI‑агентом как с “командой ролей”:
- **Router** — выбирает роль и декомпозирует запрос, если он смешанный.
- **Assistant** — превращает входящее в артефакты (задачи/проекты/заметки), поддерживает связи `[[...]]`, делает человеческие сводки.
- **Coach** — помогает формулировать цели, метрики, фокус и next actions.
- **Repo‑Maintainer** — поддерживает здоровье системы: структура, минимальный дифф, архивирование, healthcheck.

Команды для явного выбора роли и протоколы см. `00_System/Agent-Commands.md` и `00_System/Agent-Runbook.md`.

## Быстрый старт
- Откройте папку репозитория в Obsidian: `Open folder as vault`.
- Установите (минимум) плагины: **Dataview** и **Tasks**.
- Включите core‑плагин **Templates** и укажите папку шаблонов: `00_System/Templates`.
- Чтобы агент начал настройку своей части, напишите ему команду: `@Bootstrapper` (или `@Router`, если запрос смешанный).
- Начните с главных страниц: `INDEX.md`, `Personal/00_Home.md`, `Work/00_Home.md`.

## Настройка (подробно, один раз)
### 1) Git и локальная папка
- Клонируйте репозиторий и откройте его как обычную папку (Windows/WSL/macOS).
- Проверьте Git: `git status`.
- Задайте Git‑профиль (если ещё не задан):
  - `git config --global user.name "Your Name"`
  - `git config --global user.email "you@example.com"`
- Для быстрой проверки структуры запустите: `bash scripts/healthcheck.sh`

### 2) Obsidian (UI‑настройка)
- Откройте папку как vault: `Open folder as vault`.
- Включите `Templates` и укажите `00_System/Templates`.
- Установите community‑плагины:
  - `Dataview`
  - `Tasks`
- Рекомендуемые UI‑настройки (для чистого UX):
  - `Show inline title = Off`
  - `Properties in document = Collapsed` (или `Hidden`)
  - Если включены inline‑панели заметки (`Backlinks` / `Outgoing links`) — сверни их или выключи показ в документе
  - Если установлен плагин breadcrumbs/путь вверху заметки — выключи его, чтобы не видеть техназвания файлов
  - В Graph view используй сохранённый фильтр из `.obsidian/graph.json`; при необходимости дополнительно скрывай `tags`/`attachments`
- Общий граф уже настроен на скрытие тех/архивных узлов через `.obsidian/graph.json`.

### 3) Скрипты и инструменты (опционально, по мере необходимости)
- DOCX/PPTX (через `pandoc`): `scripts/setup-pandoc.sh` (если `pandoc` не установлен в системе).
- Аудио → текст: нужен `OPENAI_API_KEY` в локальном `.env` (не коммитить).
- Разбор входящих: `scripts/process-incoming.sh --domain work` (или `personal`).
- Язык тех-ID/slug для новых файлов задаётся в `00_System/Vault-Config.env` (`VAULT_TECH_ID_LANGUAGE`). В этом vault установлен `русский`.
- Для ручного создания новых сущностей (task/project/area/goal/note) используй `scripts/new-entity.sh` — он применяет `VAULT_TECH_ID_LANGUAGE` автоматически.
- Если `healthcheck` предупреждает про англоязычный slug при режиме `русский`, создавай новый файл через `scripts/new-entity.sh` (или переименуй по запросу).

## Запуск AI‑агента (быстрые сценарии)
Если роль не указана, агент сначала предложит роль и дождётся подтверждения. Для быстрого старта лучше указывать роль явно.

### Первый запуск / первичная настройка
- `@Bootstrapper Подключаюсь к vault впервые. Проведи bootstrap и проверь настройку.`

### Повседневная работа (создание/обновление артефактов)
- `@Assistant Разбери входящие и оформи заметки, потом задай уточнения.`
- `@Assistant Выдай сводку по задачам в человеческом формате.`
- `@Assistant Создай/обнови проект и связанные задачи по этому описанию: ...`

### Смешанный или неясный запрос
- `@Router Помоги выбрать роль и разложить запрос на шаги: ...`

### Поддержка репозитория и структуры
- `@Repo-Maintainer Проверь здоровье vault, синхронизируй дашборды и предложи точечные фиксы.`

Подробнее по ролям/командам: `00_System/Agent-Commands.md`, `00_System/Agent-Runbook.md`.

## Концепция (как это работает)
### 1) Артефакты и связи
- **Заметки/знания** — в `*/50_Notes/` (wiki‑ссылки `[[...]]` вместо дублей).
- **Задачи** — в `*/20_Tasks/` (1 файл = 1 задача; внутри — минимум один чекбокс `- [ ]`).
- **Проекты** — в `*/30_Projects/` (Outcome + автоматическая подборка связанных задач).
- **Цели** — в `*/60_Goals/` (метрика/горизонт + подборка проектов/задач).
- При архивировании проекта агент также проверяет связанные заметки и цель: заметки архивируются по `project:`, цель — только если у неё не остаётся активных проектов.
- Связи делаются ссылками:
  - в строке задачи — ссылки на проект/цель `[[WP-...]] [[WG-...]]` (или personal‑аналог);
  - в проекте/цели — запросы Tasks/Dataview подтягивают связанные элементы автоматически.

### 2) Дашборды
- Домашние страницы: `Personal/00_Home.md`, `Work/00_Home.md`.
- Доски задач: `*/20_Tasks/00_Taskboard.md` (Tasks‑запросы) и `*/20_Tasks/00_Kanban.md` (ручная доска ссылок).

### 3) Входящие материалы (DOCX/PPTX/аудио)
- Скидывайте файлы в `Work/00_Inbox/Incoming/` (или personal‑аналог).
- Агент конвертирует **сначала в заметки** (с `## Summary`), потом задаёт уточнения и **только после подтверждения** создаёт задачи/проекты.
- Исходники перемещаются в `*/99_Archive/Incoming/` (вместо удаления).
- Скрипт: `scripts/process-incoming.sh` (см. `Work/00_Inbox/Incoming/README.md`).

### 4) AI‑агент и роли
- Команды/маршрутизация: `00_System/Agent-Commands.md`.
- Роли/промпты: `00_System/Prompts/` (Router/Assistant/Coach/Repo‑Maintainer).
- Runbook и протоколы качества: `00_System/Agent-Runbook.md`.

## Приватность
Не храните в репозитории пароли, токены, приватные ключи и клиентские секреты. Для чувствительных данных используйте менеджер секретов или шифрование вне этого репо.
