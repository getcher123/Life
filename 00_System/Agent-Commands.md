# Команды AI‑агента (ролевая маршрутизация)

Этот файл задаёт “команды” (короткие маркеры), которые пользователь может писать в начале запроса, чтобы явно выбрать роль агента и протокол работы.

Если запрос **не содержит** команды:
1) агент **всегда** предлагает 1 наиболее релевантную роль (и 1 альтернативу, если уместно);
2) агент **ждёт подтверждения роли** от пользователя (или явного “действуй по обстоятельствам”);
3) только после этого выполняет действия по выбранной роли.

## Быстрый запуск (готовые команды пользователя)
### Первый запуск / настройка vault
- `@Bootstrapper Подключаюсь к vault впервые. Проведи bootstrap и проверь настройку.`

### Ежедневная рабочая сессия
- `@Assistant Выдай сводку по активным задачам в человеческом формате, приоритизируй по срокам и предложи фокус на сегодня.`

### Разбор входящих (без создания задач до подтверждения)
- `@Incoming Разбери входящие в Work, создай заметки с summary и задай уточняющие вопросы.`

### Планирование новой цели/проекта
- `@Plan Помоги разложить цель на проект и задачи до пятницы: ...`

### Обслуживание системы / перед коммитом
- `@Repo-Maintainer Проверь healthcheck, синхронизируй дашборды и подготовь изменения к коммиту.`

### Неясный/смешанный запрос
- `@Router Предложи роль и план выполнения для запроса: ...`

## Мини-гайд запуска агентской сессии (рекомендуемый порядок)
1. Выбери роль командой `@...` (или дай агенту предложить роль).
2. Укажи домен, если важно: `Work` / `Personal`.
3. Укажи ограничения:
   - срок (`до пятницы`, `сегодня`)
   - формат результата (`сводка`, `создай файлы`, `только план`)
   - запреты (`без коммита`, `без создания задач`, `только заметка`)
4. После изменений попроси синхронизацию дашбордов (если менялись задачи/проекты):
   - `20_Tasks/00_Taskboard.md`
   - `30_Projects/00_Projects.md`
   - `20_Tasks/00_Kanban.md`

## Роли (prompts)

### `@Router` — выбор роли и декомпозиция
- Prompt: `00_System/Prompts/Router.md`
- Когда: запрос смешанный/неясный, несколько целей.
- Результат: предложить роль(и), краткий план и начать выполнение.

### `@Assistant` — задачи/проекты/заметки
- Prompt: `00_System/Prompts/Assistant.md`
- Когда: “сделай задачи”, “оформи проект”, “разложи по файлам”, “связи wiki”.
- Результат: новые/обновлённые артефакты в нужных папках + связи `[[...]]`.

### `@Coach` — коучинг по целям
- Prompt: `00_System/Prompts/Coach.md`
- Когда: цели, мотивация, фокус, метрики, приоритеты.
- Результат: цель/проект/next actions + ритм ревью.

### `@Repo-Maintainer` — поддержка структуры/качества
- Prompt: `00_System/Prompts/Repo-Maintainer.md`
- Когда: “почини структуру”, “проверь”, “сделай healthcheck”, “прибери”.
- Результат: точечные исправления, без массовых изменений.

## Операционные команды (runbook modes)

### `@Capture` — быстрый захват
- Runbook: `00_System/Agent-Runbook.md`
- Делает: создаёт входящий файл в `Work/00_Inbox/` или `Personal/00_Inbox/` (без оптимизации).

### `@Triage` — разбор Inbox
- Runbook: `00_System/Agent-Runbook.md`
- Делает: переносит в `*/20_Tasks/`, `*/30_Projects/`, `*/50_Notes/`, `*/60_Goals/` + оставляет ссылки‑stubs.

### `@Plan` — планирование goal→project→tasks
- Runbook: `00_System/Agent-Runbook.md`
- Делает: создаёт/обновляет цель/проект/задачи, проставляет ссылки `[[...]]`, планирование в строке Tasks.

## Служебные команды (tools/workflow)

### `@Healthcheck`
- Скрипт: `bash scripts/healthcheck.sh`
- Когда: “проверь vault”, “перед коммитом”, “что сломано”.

### `@Incoming` — разбор папки входящих материалов
- Док: `Work/00_Inbox/Incoming/README.md`
- Скрипт: `scripts/process-incoming.sh --domain work|personal`
- Делает: создаёт заметки из DOCX/аудио и переносит исходники в `*/99_Archive/Incoming/` (задачи/проекты — только после подтверждения).

### `@DOCX`
- Док: `00_System/Docx-Tools.md`
- Делает: DOCX → заметка (и `## Summary`), затем вопросы; задачи/проекты только после подтверждения.

### `@Audio`
- Скрипт: `scripts/audio2note.sh`
- Делает: audio → заметка (и `## Summary`), затем вопросы; задачи/проекты только после подтверждения.

### `@GoogleSheets`
- Док: `00_System/Google-Sheets-Tools.md`
- Скрипт: `scripts/gsheets_fetch.py`
- Делает: читает Google‑таблицы через service account (креды локально, не в Git).
