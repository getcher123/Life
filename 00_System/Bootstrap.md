# Bootstrap (автоподготовка агента при первом подключении)

Цель: чтобы при первом подключении к репозиторию агент сам проверил всё критичное, помог пользователю сделать ручные шаги (где требуется UI), и дальше поддерживал работоспособность системы.

## Как запускать
- Если ты человек: открой `INDEX.md` → `00_System/Bootstrap.md` и следуй шагам.
- Если ты агент: используй промпт `00_System/Prompts/Bootstrapper.md`.
- Если вы работаете через роли: команда запуска для пользователя — `@Bootstrapper`.

## Что подготовить до bootstrap (чек-лист)
- Репозиторий открыт локально и доступен для записи.
- Папка открывается в Obsidian как vault.
- Пользователь готов сделать UI‑шаги в Obsidian (Templates / Dataview / Tasks).
- (Опционально) Git remote/репозиторий уже создан, если хотите сразу настроить синхронизацию.
- (Опционально) `.env` подготовлен для инструментов, которым нужны ключи (например, audio → text).
- (Рекомендуется) В `00_System/Vault-Config.env` проверен язык tech-ID/slug (`VAULT_TECH_ID_LANGUAGE`); для этого vault ожидается `русский`.

## Детектор “первого запуска”
Считаем, что это первый запуск, если выполняется хотя бы одно:
- нет `.obsidian/` или он пустой;
- не установлены плагины Dataview/Tasks (пользователь ещё не включал их в Obsidian);
- нет ни одной задачи/проекта/цели (система ещё не “ожила”);
- Git не настроен (нет `user.name`/`user.email`) или нет первого коммита.

## Шаги bootstrap (что агент делает сам)
1. Проверяет структуру и наличие ключевых файлов/папок (см. `scripts/healthcheck.sh`).
2. Проверяет настройки Obsidian, которые хранятся в репо (например `.obsidian/templates.json`).
3. Проверяет, что шаблоны и дашборды на месте (Templates/Taskboard/Projects/Goals/Reviews).
4. Проверяет `00_System/Vault-Config.env` и напоминает, что ручное создание сущностей нужно делать через `scripts/new-entity.sh`.
5. Создаёт “первую живую связку” (по согласованию):
   - 1 area, 1 goal, 1 project, 3–5 tasks (work или personal).
6. Подготовит Git к работе:
   - подскажет пользователю задать `user.name`/`user.email`;
   - предложит первый коммит и (опционально) remote.

## Шаги bootstrap (что требуется от пользователя)
В Obsidian есть действия через UI — агент не может кликать вместо пользователя:
1. Открыть папку как vault.
2. Включить Templates и указать папку `00_System/Templates`.
3. Установить community‑плагины Dataview и Tasks.
4. (Рекомендуется) Выключить `Show inline title` и свернуть `Properties` для более чистого UX.
5. (Рекомендуется) Свернуть/выключить inline `Backlinks` / `Outgoing links`, а также breadcrumbs (если включены), чтобы убрать техмусор.

## Проверка “готово”
Bootstrap завершён, если:
- `scripts/healthcheck.sh` завершается без ошибок;
- в Obsidian работают страницы:
  - `Personal/20_Tasks/00_Taskboard.md` и/или `Work/20_Tasks/00_Taskboard.md`;
  - `*/30_Projects/00_Projects.md`, `*/60_Goals/00_Goals.md`, `*/70_Reviews/00_Reviews.md`;
- есть минимум 1 цель, 1 проект, 1 задача (и задача подтягивается на странице проекта через Tasks‑запрос).

## Команды запуска после bootstrap (agent startup guides)
### Базовый старт рабочей сессии
- `@Assistant Выдай сводку по активным задачам в человеческом формате и предложи фокус на сегодня`

### Разбор входящих материалов
- `@Incoming Разбери входящие в Work, создай заметки и summary, задачи не создавай без подтверждения`

### Планирование (goal → project → tasks)
- `@Plan Разложи эту цель на проект и задачи с дедлайнами до пятницы: ...`

### Проверка здоровья системы перед коммитом
- `@Repo-Maintainer Проверь healthcheck, синхронизируй taskboard/projects/kanban и подготовь коммит`

### Если непонятно, с чего начать
- `@Router Предложи роль для этого запроса и краткий план: ...`

Полный список ролей и команд: `00_System/Agent-Commands.md`.

## Ожидаемый результат bootstrap (что должно появиться)
- Рабочий Obsidian‑vault с включёнными `Templates`, `Dataview`, `Tasks`.
- Настроенный “чистый” UI в Obsidian (без лишних техпанелей/заголовков в заметках).
- Рабочие точки входа без ошибок: `00_Home`, `00_Taskboard`, `00_Projects`, `00_Goals`, `00_Reviews`.
- Минимум одна “живая связка” (`goal → project → tasks`) в `Work` или `Personal`.
- Базовая агентная схема запуска: пользователь знает команды `@Bootstrapper`, `@Assistant`, `@Router`, `@Repo-Maintainer`.
- Пользователь/агент знают, что новые сущности вручную создаются через `scripts/new-entity.sh` с учётом `VAULT_TECH_ID_LANGUAGE`.

## Дальнейшая поддержка
- На каждой “сессии” агент выполняет лёгкий healthcheck (см. `00_System/Support.md`).
- Раз в неделю агент инициирует review по шаблону и помогает держать связи goal↔project↔task.
